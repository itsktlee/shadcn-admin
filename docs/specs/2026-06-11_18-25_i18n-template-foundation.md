# Spec: i18n-template-foundation

## Goal
- 要解决什么问题：为模板接入正式 `i18next` 多语言基础层，而不是停留在纯中文硬编码。
- 最终目标：形成可长期复用的模板级中英双语基础设施，并让设置页的语言选项真正控制全局语言。
- 本轮核心目标：补齐 auth 页、设置页语言信息架构、顶栏语言双入口，以及 tasks / users / data-table 共享层剩余硬编码文案。
- 验收结果：已完成本轮目标。

## Done Contract
- 什么算完成：项目内具备可运行的 `i18next` 基础层，默认中文、支持英文、英文缺词回退中文；auth 页、tasks / users 页面和 data-table 共享层完成接线；设置页出现独立“语言”分区；顶栏提供语言快捷入口。
- 由什么证明：源码静态检查、关键文件 diff、语言切换链路代码可追踪、本地页面中英切换验收。
- 哪些情况仍算未完成：auth、tasks、users 或共享表格控件仍有明显英文残留；语言仍留在账户页；或设置页与顶栏入口状态不同步。

## Scope
- In: `src/i18n` 资源扩展，auth 页面与表单、设置页“语言”独立分区、顶栏语言切换组件、模板共享控件、Clerk 示例页，以及 tasks / users 页表格、抽屉、批量操作和演示数据标签。
- Out: 全仓库所有业务页的完整翻译、自动翻译流程、提交 git。
- 用户已切分的任务单元：正式模板级 `i18next` 接入后的第二轮模板收口。
- 轻量评估：足够小
